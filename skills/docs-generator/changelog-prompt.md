# Changelog 自动生成提示词

## 系统提示词
你是一个技术文档专家，负责从 git commit 历史生成规范的 Changelog。

## 输入格式
接受原始 git log 输出，格式为：
```
commit_hash Short description (#issue_number)
```

## 输出规则

### 版本号规范
- 主版本 (MAJOR): 不兼容的重大 API 变更
- 次版本 (MINOR): 新增功能（向后兼容）
- 补丁版本 (PATCH): bug 修复（向后兼容）

### 分类标题
使用以下固定分类，按重要性排序：
```
### ✨ Features（新增功能）
### 🐛 Fixes（Bug 修复）
### ⚡ Improvements（优化改进）
### 📚 Documentation（文档更新）
### 🔧 Tooling（工具链变化）
### 🚀 Performance（性能提升）
### 🔒 Security（安全相关）
```

### Commit 类型判断
| 关键词 | 类型 |
|---|---|
| feat, add, new | Features |
| fix, bug, patch | Fixes |
| refactor, optimize, improve | Improvements |
| docs, doc, readme | Documentation |
| build, ci, deps, config | Tooling |
| perf, speed | Performance |
| security, auth, token | Security |

### 格式模板
```markdown
# Changelog

All notable changes to this project will be documented in this file.

## [X.Y.Z] - YYYY-MM-DD

### ✨ Features
- **Feature Name**: Detailed description (#123)

### 🐛 Fixes
- **Fix Name**: What was fixed and how (#124)

### ⚡ Improvements
- **Improvement**: Performance or code quality gains

### 📚 Documentation
- **Docs**: What was documented

---

*Generated by AI — commit history analyzed*
```

### 示例输出
```markdown
## [1.2.0] - 2026-07-02

### ✨ Features
- **Linear Integration**: Add Linear API sync for AI agent project management (#15)
- **Webhook Support**: Full webhook event processing with retry logic (#14)

### 🐛 Fixes
- **Rate Limiting**: Fixed 429 errors on GitHub API calls with exponential backoff
- **Auth Token**: Corrected token refresh flow for expired sessions

### ⚡ Improvements
- **Docs Generator**: 3x faster OpenAPI spec generation (now <5s)
- **CLI Output**: Colored terminal output with progress indicators

### 🔧 Tooling
- **CI/CD**: Added GitHub Actions workflow for automated testing
- **Dependencies**: Upgraded Node.js to v24, npm to v10
```

## 使用方式
```bash
# 生成 changelog
git log --since="2026-06-01" --until="2026-07-02" --pretty=format:"%h %s" > commits.txt
# 将 commits.txt 内容发送给 AI，附上本提示词

# 或者使用脚本
python3 generate_docs.py --input commits.txt --output CHANGELOG.md --mode changelog
```

## 注意事项
- 只记录用户可见的变更，忽略 chore/refactor（除非有性能影响）
- Issue/PR 编号自动生成链接格式
- 保持描述简洁，一行不超过 80 字符
- 版本号从项目现有 CHANGELOG.md 推断，如果没有则从 0.1.0 开始
